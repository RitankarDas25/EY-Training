list of important Git commands with their use:

1. git init – Initializes a new Git repository in your project directory.
2. git clone <repo_url>– Clones a remote repository to your local machine.
3. git status – Shows the current state of the working directory and staging area.
4. git add <file> – Adds file(s) to the staging area.
5. git commit -m "message" – Commits staged changes with a message.
6. git push – Pushes local commits to the remote repository.
7. git pull – Fetches and merges changes from the remote repository.
8. git fetch – Downloads changes from the remote without merging.
9. git merge <branch> – Merges the specified branch into the current one.
10. git branch – Lists all local branches.
11. git branch <name> – Creates a new branch.
12. git checkout <branch> – Switches to the specified branch.
13. git checkout -b <branch> – Creates and switches to a new branch.
14. git log – Shows the commit history.
15. git remote -v – Lists remote repositories and their URLs.
16. git reset <file>– Unstages a file from the staging area.
17. git reset --hard <commit> – Resets working directory and index to a specific commit.
18. git revert <commit> – Reverts a specific commit with a new commit.
19. git stash– Temporarily saves changes not yet committed.
20. git stash pop – Applies and removes the last stashed changes.
21. git diff– Shows changes between working directory and index or commits.
22. git tag <name> – Creates a tag (usually for releases).
23. git config – Sets Git configuration values (e.g., username, email).
24. git rm <file>– Removes a file from the working directory and staging area.
25. git show <commit>– Shows details about a specific commit.
